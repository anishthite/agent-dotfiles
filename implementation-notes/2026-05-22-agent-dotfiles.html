agent-dotfiles — initial scaffold

Started: 2026-05-22 · Workspace: ~/workspace/agent-dotfiles · Status: scaffolded, awaiting user decisions before first sync + push.

Goal

Single git repo that mirrors the user's per-agent dotdirs (~/.agents/, ~/.claude/, ~/.codex/, ~/.pi/, ~/.gemini/), tracks all authored skills and settings, excludes sessions/auth/caches, and can be cloned + bootstrapped on a fresh machine.

Decisions

IDDecisionRationale
D-1 Mirror per-vendor dir structure inside repo (literal .agents/, .claude/, …). User asked for this explicitly. Matches mental model. Bootstrap is a straight reverse-rsync.
D-2 Use rsync for both directions, gated by .gitignore via --filter=':- .gitignore'. No new tool to install (rsync ubiquitous). Self-documenting via gitignore. Simpler than chezmoi/stow for a single-user, mirror-style repo.
D-3 Preserve symlinks as-is in repo (rsync -a default), do not follow them. Repo content stays small; symlink targets reproduce the canonical-plus-fanout pattern when bootstrapped. Bootstrap also recreates symlinks fresh after rsync as a safety net.
D-4 Gitignore .codex/rules/default.rules by default. File contains historical "always allow" command rules with plaintext API keys (Anthropic, PostHog, Google Maps, Supabase DB URL). Even private repos shouldn't carry these. User must sanitize + rotate before opting in.
D-5 Gitignore .claude/settings.local.json by default. Convention: *.local.* files are machine-local overrides. User can comment out the rule if they want to track.
D-6 Bootstrap defaults to dry-run; requires --apply to write. Restoring config into $HOME is high-blast-radius. Force the user to see diff first.
D-7 Repo dir: ~/workspace/agent-dotfiles (working assumption). Matches existing ~/workspace/ convention. User can rename.

Tradeoffs

IDTradeoff
T-1 Mirror-style repo (D-1) makes git diff readable but couples repo layout to current vendor dirs. If pi/Codex moves their config home, sync.sh needs an edit. Acceptable: vendor home dirs are slow-moving.
T-2 Preserving symlinks (D-3) keeps repo small but absolute-path symlinks (e.g. ~/.pi/agent/skills/excalidraw-diagram → /Users/anishthite/.agents/skills/excalidraw-diagram) break on different $HOME. Mitigation: bootstrap step 2 recreates all skill symlinks fresh from ~/.agents/skills/ with computed relative paths, regardless of what was stored.

Reversibility

IDNote
R-1Repo creation is fully reversible (delete ~/workspace/agent-dotfiles).
R-2sync.sh only writes inside the repo, never touches $HOME.
R-3bootstrap.sh --apply writes into $HOME. Mitigation: rsync uses --no-perms and never --delete (so existing files outside the repo's scope are not removed). Dry-run is default.
R-4GitHub push is reversible (gh repo delete); not so for secrets if visibility was wrong on push. Hence Q-2/Q-3 are blocking.
